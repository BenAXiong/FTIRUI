A few thoughts:

The runtime directory is large but logical; when adding features it’s worth keeping that facade approach so orchestration doesn’t bloat. Maybe consider surfacing a quick index (e.g., README linking to each facade) for future contributors.
The legacy initControls.js still mixes API calls, DOM tweaks, and Plotly hooks. As workspace panels mature, migrating those responsibilities into the runtime (or killing the old preview panel) will simplify things.
The template split between tabbed vs. standalone works, but note that any header/back-button tweaks live outside the workspace include. If you ever add workspace-specific controls there, documenting that dependency will help avoid confusion.
